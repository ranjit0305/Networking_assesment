# LAN Simulation and Ethernet Frame Analysis

## Objective
The objective of this experiment is to simulate a small LAN network using Cisco Packet Tracer and analyze Ethernet frames using Wireshark. The experiment demonstrates MAC address learning, ICMP communication, and packet structure.

---

# Network Topology

A simple LAN network was created using:
- 2 PCs
- 1 Switch (Cisco 2960)

Both PCs were connected to the switch using Ethernet cables.

# Ethernet Frame Structure

Each Ethernet frame contains the following components:

| Field | Description |
|-----|-----|
| Destination MAC Address | Receiver device MAC |
| Source MAC Address | Sender device MAC |
| EtherType | Identifies the protocol (IPv4) |
| Payload | Actual data |
| FCS | Error detection (not visible in Wireshark) |

# Observations

1. The switch dynamically learns MAC addresses from incoming frames.
2. ICMP packets are used for connectivity testing.
3. Wireshark allows inspection of packet headers and payloads.
4. Ethernet frames encapsulate IP packets for transmission over the network.
5. FCS is normally not visible in Wireshark because network cards remove it before capture.

---

# Conclusion

A simple LAN was successfully simulated using Cisco Packet Tracer. Network connectivity was verified using the ping command. Wireshark was used to capture and analyze Ethernet frames and ICMP packets. The experiment helped understand MAC addressing, frame structure, and packet flow in a LAN environment.
